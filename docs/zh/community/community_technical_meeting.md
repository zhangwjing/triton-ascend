# 技术例会

Triton-Ascend 定期召开社区技术会议，作为项目治理的核心决策机制。技术方向、重大提案评审和版本发布等重大事项在技术会议中讨论和决定，由全体 `Maintainer` 投票。会议对所有社区成员开放，任何人均可参与讨论，但决策和投票仅由 `Maintainer` 进行。

## 会议内容

- 技术方向讨论与决策
- 重大提案（RFC）评审与投票
- 版本发布规划与时间线
- 社区 Issue 和 PR 优先级排序
- 贡献者提名与移除讨论

## 投票规则

- 未出席会议的 `Maintainer` 须在会议后 3 个工作日内提交投票；逾期未投票视为弃权
- 投票通过条件为参与投票的 `Maintainer` 超过半数同意（弃权不计入投票总数）
- 投票结果记录于会议纪要中

## 决策流程

### 常规决策

日常开发中的常规决策（如 `Bug` 修复、小功能改进）由相关模块的 `Maintainer` 在 `Issue` 或 `PR` 中审阅决定，无需召开会议。

### 重大决策

以下事项视为重大决策，需由 `Maintainer` 在社区技术会议中集体讨论：

- 重大架构变更
- 破坏性 `API` 变更
- 新增核心功能或移除现有功能
- 第三方依赖的重大升级
- 版本发布规划

重大决策流程：

1. 提交 `RFC`（征求意见稿），描述变更内容、影响范围和实施方案
2. 在技术会议中讨论
3. `Maintainer` 共同决策
